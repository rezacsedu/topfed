Data configuration

back

Federation members can be added to a federation either directly as a list of endpoints, or using a data configuration file via the dataConfig property (see Java examples). In a data configuration the federation members are specified in a turtle syntax.

Example 1: SPARQL Federation:

@prefix fluid: <http://fluidops.org/config#>.

<http://DBpedia> fluid:store "SPARQLEndpoint";
fluid:SPARQLEndpoint "http://dbpedia.org/sparql".

<http://NYtimes> fluid:store "SPARQLEndpoint";
fluid:SPARQLEndpoint "http://api.talis.com/stores/nytimes/services/sparql".

<http://LinkedMDB> fluid:store "SPARQLEndpoint";
fluid:SPARQLEndpoint "http://data.linkedmdb.org/sparql".

Example 2: SPARQL Federation with Sesame remote repositories

<http://dbpedia> fluid:store "RemoteRepository";
fluid:repositoryServer "http://<host>/openrdf-sesame" ;
fluid:repositoryName "<repoName>" .

Example 3: Local Federation (NativeStore):

@prefix fluid: <http://fluidops.org/config#>.

<http://DBpedia> fluid:store "NativeStore";
fluid:RepositoryLocation "repositories\\native-storage.dbpedia36".

<http://NYTimes> fluid:store "NativeStore";
fluid:RepositoryLocation "repositories\\native-storage.nytimes".
Note that also hybrid combinations are possible.
